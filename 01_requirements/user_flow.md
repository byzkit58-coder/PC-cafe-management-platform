# 사용자 플로우 정의

## 1. 문서 목적

본 문서는 PC방 매장 운영 플랫폼의 주요 사용자 흐름을 정의한다.

본 포트폴리오에서는 모든 기능을 동일한 깊이로 다루지 않고, 실제 운영 리스크가 큰 핵심 플로우를 중심으로 테스트를 설계한다.

---

# 2. 핵심 플로우 1: 로그인 및 좌석 사용 시작

## 2.1 플로우 설명

일반 사용자가 로그인 후 사용 가능한 좌석을 선택하고, 좌석 상태가 사용 중으로 변경되는 흐름이다.

## 2.2 기본 흐름

```text
사용자 로그인
→ 좌석 현황 화면 진입
→ 사용 가능한 좌석 선택
→ 좌석 사용 시작 버튼 클릭
→ 좌석 상태가 '사용 가능'에서 '사용 중'으로 변경
→ 관리자 화면에서도 해당 좌석이 '사용 중'으로 표시
```

# 3. 핵심 플로우 2: 음식 주문

## 3.1 플로우 설명

좌석을 사용 중인 사용자가 음식 메뉴를 선택하고 주문하면, 매장 관리자 화면에 주문이 접수되는 흐름이다.

## 3.2 기본 흐름

```text
사용자 로그인
→ 좌석 사용 시작
→ 음식 메뉴 화면 진입
→ 메뉴 선택
→ 쿠폰 또는 포인트 적용
→ 주문하기 클릭
→ 결제 진행
→ 주문 생성
→ 관리자 주문 목록에 신규 주문 표시
```

## 3.3 주문 상태 흐름

```text
주문 생성
→ 결제대기
→ 결제완료
→ 접수
→ 조리중
→ 완료
```

# 4. 핵심 플로우 3: 결제 실패 / 취소 / 쿠폰 복원

## 4.1 플로우 설명

사용자가 결제를 시도했으나 실패하거나, 결제 완료 후 주문을 취소하는 상황에서 주문, 결제, 쿠폰, 포인트 상태가 일관되게 유지되는지 확인하는 흐름이다.

## 4.2 결제 실패 흐름

```text
주문 생성
→ 결제 시도
→ 결제 실패
→ 주문 상태는 '결제대기' 유지
→ 쿠폰/포인트는 사용 처리되지 않음
→ 정산에는 반영되지 않음
```

## 4.3 주문 취소 흐름

```text
결제 완료 주문 생성
→ 쿠폰 사용
→ 주문 취소 요청
→ 주문 상태 '취소완료' 변경
→ 사용 쿠폰 복원
→ 사용 포인트 복원
→ 정산 금액에서 제외
```

# 5. 핵심 플로우 4: 관리자 주문 처리

## 5.1 플로우 설명

사용자가 주문한 음식이 매장 관리자 화면에 노출되고, 매장 직원 또는 매장 관리자가 주문 상태를 변경하는 흐름이다.

## 5.2 기본 흐름

```text
사용자 주문 생성
→ 관리자 주문 목록에 주문 표시
→ 관리자 주문 접수
→ 주문 상태 '접수' 변경
→ 관리자 '조리중' 변경
→ 관리자 '완료' 변경
→ 사용자 주문 내역에 완료 상태 표시
→ 완료 주문이 정산에 반영
```

# 6. 핵심 플로우 5: 권한 별 접근 제어

## 6.1 플로우 설명

사용자 권한에 따라 접근 가능한 메뉴와 데이터 범위가 제한되는지 확인하는 흐름이다.

## 6.2 기본 검증 흐름

```text
일반 사용자 로그인
→ 관리자 URL 직접 접근
→ 접근 차단

매장 A 관리자 로그인
→ 매장 A 주문 조회
→ 조회 성공

매장 A 관리자 로그인
→ 매장 B 주문 상세 URL 직접 접근
→ 접근 차단

본사 관리자 로그인
→ 전체 매장 정산 조회
→ 조회 성공
```

# 7. 핵심 플로우 6: 정산 반영

## 7.1 플로우 설명

완료된 주문만 매출 및 정산 금액에 반영되는지 확인하는 흐름이다.

## 7.2 기본 흐름

```text
사용자 주문 생성
→ 결제 완료
→ 관리자 주문 완료 처리
→ 주문 상태 완료
→ 일 매출 합계에 반영
→ 매장 관리자 정산 화면에서 확인
```
