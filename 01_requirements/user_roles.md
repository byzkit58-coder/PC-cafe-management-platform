# 사용자 권한 정의

## 1. 권한 정의 목적

본 문서는 PC방 매장 운영 플랫폼에서 사용하는 권한별 역할과 접근 가능 기능을 정의한다.

권한별 접근 제어는 본 포트폴리오의 핵심 검증 영역 중 하나이다.  
특히 일반 사용자, 매장 직원, 매장 관리자, 본사 관리자 간 접근 가능한 데이터와 기능이 다르기 때문에, 화면 접근뿐 아니라 API 직접 호출 시에도 권한 검증이 필요하다.

## 4. 권한별 상세 설명

### 4.1 일반 사용자

일반 사용자는 PC방 좌석을 선택하고, 좌석 사용 중 음식 주문과 결제를 진행할 수 있다.

#### 가능 기능

- 로그인
- 사용 가능한 좌석 선택
- 본인 좌석 사용 상태 확인
- 음식 메뉴 조회
- 음식 주문
- 쿠폰 사용
- 결제
- 본인 주문 내역 조회
- 본인 주문 취소

#### 제한 기능

- 관리자 페이지 접근 불가
- 다른 사용자의 주문 내역 조회 불가
- 주문 상태 직접 변경 불가
- 정산 정보 조회 불가
- 쿠폰 정책 변경 불가

---

### 4.2 매장 직원

매장 직원은 특정 매장에 접수된 주문을 확인하고, 주문 상태를 변경할 수 있다.

#### 가능 기능

- 매장 주문 목록 조회
- 주문 접수 처리
- 주문 상태 변경
- 좌석 상태 조회
- 재고 상태 조회

#### 제한 기능

- 매출/정산 상세 조회 불가
- 직원 관리 불가
- 쿠폰 정책 변경 불가
- 다른 매장 주문 조회 불가

---

### 4.3 매장 관리자

매장 관리자는 담당 매장의 운영 정보를 관리한다.

#### 가능 기능

- 담당 매장 주문 조회
- 담당 매장 주문 상태 변경
- 담당 매장 좌석 상태 관리
- 담당 매장 매출 조회
- 담당 매장 정산 조회
- 담당 매장 직원 관리

#### 제한 기능

- 다른 매장 주문 조회 불가
- 다른 매장 정산 조회 불가
- 전체 쿠폰 정책 관리 불가
- 본사 관리자 메뉴 접근 불가

---

### 4.4 본사 관리자

본사 관리자는 전체 매장 현황과 정책을 관리한다.

#### 가능 기능

- 전체 매장 목록 조회
- 전체 매장 주문 현황 조회
- 전체 매장 매출/정산 조회
- 쿠폰 정책 관리
- 통합 리포트 확인

#### 제한 기능

- 일반 사용자 좌석 선택 불가
- 일반 사용자 주문 생성 불가
- 매장 직원처럼 개별 주문 조리 상태를 직접 처리하지 않음

## 5. 권한 검증 주요 리스크

| 리스크                           | 설명                                             | 영향                       |
| -------------------------------- | ------------------------------------------------ | -------------------------- |
| 일반 사용자의 관리자 페이지 접근 | URL 직접 입력으로 관리자 페이지 접근 가능        | 개인정보/주문정보 노출     |
| 매장 간 데이터 접근              | 매장 A 관리자가 매장 B 주문 조회                 | 운영 데이터 유출           |
| API 권한 검증 누락               | 화면에서는 차단되지만 API 직접 호출 가능         | 보안 취약점                |
| 권한 변경 후 세션 미반영         | 권한이 변경되었지만 기존 세션에서 계속 접근 가능 | 권한 관리 오류             |
| 메뉴 노출 오류                   | 접근 불가 기능이 화면에 노출                     | 사용자 혼란 및 보안 리스크 |

## 6. 권한 검증 기준

권한 검증은 화면 접근과 API 접근을 모두 포함해야 한다.

### 화면 접근 기준

- 접근 권한이 없는 메뉴는 노출되지 않아야 한다.
- URL을 직접 입력해도 접근이 차단되어야 한다.
- 접근 차단 시 적절한 오류 페이지 또는 권한 없음 메시지가 표시되어야 한다.

### API 접근 기준

- 권한이 없는 사용자가 API를 호출하면 401 또는 403 응답이 반환되어야 한다.
- 매장 관리자는 본인이 담당하는 매장의 데이터만 조회할 수 있어야 한다.
- 일반 사용자는 본인의 주문 데이터만 조회할 수 있어야 한다.
