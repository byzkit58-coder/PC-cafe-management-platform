# QA 리뷰 후 보완 테스트케이스

## 1. 문서 목적

본 문서는 AI가 생성한 테스트케이스 초안을 QA 관점에서 검토하고 보완한 결과를 정리한다.

AI 생성 결과는 기본적인 정상 케이스와 일반 예외 케이스는 포함했지만, 운영 플랫폼에서 중요한 데이터 정합성, 권한 분리, 상태 전이 제한, 정산 반영 조건 일부가 부족했다.

따라서 본 문서에서는 AI 결과를 기반으로 QA가 추가하거나 수정한 테스트케이스를 정의한다.

---

## 2. QA 리뷰 기준

AI 생성 테스트케이스는 다음 기준으로 검토했다.

| 검토 기준          | 설명                                                  |
| ------------------ | ----------------------------------------------------- |
| 요구사항 충족 여부 | 1단계 요구사항을 충분히 반영했는지 확인               |
| 리스크 반영 여부   | 2단계 High 리스크 영역이 포함되었는지 확인            |
| 데이터 정합성      | 주문, 결제, 쿠폰, 포인트, 정산 상태가 일관되는지 확인 |
| 권한 검증 수준     | 화면 접근뿐 아니라 API 접근까지 검증하는지 확인       |
| 상태 전이 검증     | 비정상 상태 변경 차단 케이스가 포함되었는지 확인      |
| 자동화 적합성      | E2E/API/Manual 분류가 타당한지 확인                   |

---

## 3. AI 결과에서 보완이 필요한 부분

| 구분                | AI 결과의 한계             | QA 보완 방향                              |
| ------------------- | -------------------------- | ----------------------------------------- |
| 권한 검증           | 화면 접근 중심으로 작성됨  | API 직접 호출 권한 검증 추가              |
| 매장 간 데이터 분리 | 주문 접근 차단만 일부 포함 | 주문/정산 데이터 모두 분리 검증           |
| 결제 실패           | 주문 상태 유지 검증은 포함 | 쿠폰/포인트 미사용 유지, 정산 미반영 추가 |
| 정산                | 완료/취소 주문 중심        | 결제실패, 환불완료, 매장별 분리 추가      |
| 상태 전이           | 정상 상태 변경 중심        | 결제대기 → 완료 직접 변경 차단 추가       |
| 중복 처리           | 중복 결제는 포함           | 중복 주문 생성 방지 추가                  |
| 쿠폰/포인트         | 취소 후 복원 포함          | 결제 실패 시 사용 처리 방지 추가          |
| 좌석                | 사용 중 좌석 차단 포함     | 동시 선택 요청 1건만 성공 케이스 추가     |
