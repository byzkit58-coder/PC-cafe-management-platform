# 릴리즈 체크리스트

## 1. 문서 목적

본 문서는 PC방 매장 운영 플랫폼의 배포 전 확인해야 할 핵심 회귀 테스트 항목을 정의한다.

릴리즈 체크리스트는 전체 테스트케이스 중 운영 영향도가 크고 반복 검증 가치가 높은 항목을 선별하여 구성한다.

본 체크리스트는 Playwright 자동화 대상 선정과 CI 실행 기준으로 활용할 수 있다.

---

## 2. 릴리즈 전 핵심 확인 기준

배포 전 다음 영역은 반드시 확인한다.

| 영역 | 확인 이유 |
|---|---|
| 로그인 | 모든 기능의 사전 조건 |
| 좌석 선택 | 서비스 이용 시작 흐름 |
| 음식 주문 | 핵심 사용자 행동 |
| 관리자 주문 처리 | 실제 매장 운영과 연결 |
| 결제 실패 처리 | 금전 데이터 정합성 |
| 중복 결제 방지 | 사용자 피해 방지 |
| 쿠폰/포인트 복원 | 사용자 혜택 데이터 정합성 |
| 권한 제어 | 운영 데이터 보호 |
| 정산 반영 | 매출 데이터 신뢰성 |

## 4. 릴리즈 승인 기준

릴리즈 승인 기준은 다음과 같다.

| 구분 | 기준 |
|---|---|
| 필수 항목 | 모두 Pass 필요 |
| 권장 항목 | 실패 시 영향도 판단 후 릴리즈 여부 결정 |
| High 우선순위 TC | 실패 시 릴리즈 보류 |
| 결제/정산/권한 관련 TC | 실패 시 릴리즈 불가 |
| Medium 우선순위 TC | 우회 방안 또는 수정 계획 필요 |
| Low 우선순위 TC | 이슈 등록 후 릴리즈 가능 |

## 5. 릴리즈 보류 기준

다음 문제가 발생하면 릴리즈를 보류한다.

| 보류 기준 | 사유 |
|---|---|
| 중복 결제 발생 | 사용자 금전 피해 가능 |
| 결제 실패 주문이 정산 반영 | 매출 데이터 오류 |
| 주문 취소 후 쿠폰/포인트 미복원 | 사용자 혜택 손실 |
| 매장 간 데이터 접근 가능 | 운영 데이터 유출 |
| 일반 사용자의 관리자 API 접근 가능 | 권한 보안 결함 |
| 완료 주문 정산 미반영 | 매출 누락 |
| 사용자 주문이 관리자 화면에 미표시 | 실제 주문 처리 불가 |

## 9. 릴리즈 체크리스트 결론

본 릴리즈 체크리스트는 PC방 매장 운영 플랫폼의 핵심 운영 리스크를 기준으로 구성되었다.

특히 결제, 정산, 쿠폰/포인트, 권한, 주문 연동 영역은 장애 발생 시 사용자 피해와 운영 데이터 오류로 이어질 수 있으므로 필수 확인 대상으로 분류한다.

본 체크리스트는 이후 Playwright 자동화와 GitHub Actions 기반 회귀 테스트 구성의 기준으로 활용한다.
