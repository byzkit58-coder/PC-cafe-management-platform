# 테스트 시나리오

## 1. 문서 목적

본 문서는 PC방 매장 운영 플랫폼의 핵심 테스트 시나리오를 정의한다.

테스트 시나리오는 1단계 요구사항과 2단계 리스크 분석을 기반으로 작성되었으며, 이후 상세 테스트케이스 매트릭스와 Playwright 자동화 대상 선정의 기준으로 사용된다.

본 프로젝트는 단순 기능 확인보다 실제 매장 운영에 영향을 줄 수 있는 리스크를 중심으로 테스트 시나리오를 구성한다.

---

## 2. 테스트 시나리오 설계 기준

본 프로젝트의 테스트 시나리오는 다음 기준으로 설계한다.

| 기준 | 설명 |
|---|---|
| 운영 영향도 | 실제 매장 운영에 장애를 줄 수 있는 흐름 우선 |
| 데이터 정합성 | 주문, 결제, 쿠폰, 포인트, 정산 상태 일관성 검증 |
| 권한 통제 | 사용자별, 매장별 접근 제한 검증 |
| 예외 처리 | 결제 실패, 품절, 중복 클릭, 취소 상황 검증 |
| 자동화 가능성 | 반복 검증 가치가 높은 케이스 우선 선정 |

## 4. 시나리오 상세

### SCN-001. 로그인 및 좌석 선택

#### 목적

일반 사용자가 정상 로그인 후 사용 가능한 좌석을 선택하고, 좌석 상태가 사용 중으로 변경되는지 확인한다.

#### 주요 흐름

```text
사용자 로그인
→ 좌석 현황 화면 진입
→ 사용 가능한 좌석 선택
→ 좌석 사용 시작
→ 좌석 상태 '사용 가능'에서 '사용 중'으로 변경
→ 관리자 화면에도 동일 상태 반영
